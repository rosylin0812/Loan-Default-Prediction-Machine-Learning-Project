## Loan Default Risk Analysis
Loan Default Risk Analysis is a data analytics and machine learning project that explores borrower characteristics and predicts the likelihood of loan default. 
Using real-world lending data, the project focuses on data cleaning, exploratory data analysis (EDA), 
feature selection, and building a logistic regression model to understand key drivers of default risk.

The goal of this project is not only prediction accuracy, 
but also business interpretability—identifying which borrower attributes most strongly influence default outcomes and 
how such insights could support credit risk decision-making.

## Project Overview

This project analyzes loan application data to understand the factors that contribute to loan default and 
to build a predictive model for default risk. The analysis combines data cleaning, exploratory data analysis (EDA), 
and machine learning to provide both predictive results and business-relevant insights.

The project is designed to mirror how credit risk or risk analytics teams approach real-world default prediction problems.

## Objectives

Clean and prepare loan application data for analysis

Explore borrower demographics and financial attributes

Identify key factors associated with loan default

Build and evaluate a logistic regression model to predict default risk

Translate model results into interpretable, business-friendly insights

## Dataset

The dataset contains borrower and loan-level information, including:

Applicant income and employment details

Credit history and loan attributes

Demographic variables

Target variable indicating loan default

## Methodology
1. Data Cleaning

Checked for missing values and data inconsistencies

Handled null values through removal or imputation where appropriate

Ensured data types were suitable for analysis and modeling

2. Exploratory Data Analysis (EDA)

Analyzed distributions of key financial and demographic variables

Compared default vs. non-default borrowers

Visualized correlations and trends to identify risk patterns

3. Feature Selection

Examined relationships between features and default status

Removed highly correlated or low-impact variables

Selected features that improved interpretability and model stability

4. Modeling

Built a logistic regression model to predict loan default

Split data into training and testing sets

Evaluated model performance using accuracy and classification metrics
